# Project Overview

This project demonstrates the implementation of **AWS Identity and Access Management (IAM)** to securely control access to AWS resources following the **Principle of Least Privilege**.

The lab focuses on creating IAM users, user groups, managed policies, custom policies, and IAM roles while validating permissions through real-world access testing.

The objective is to understand how AWS secures cloud environments by granting users only the permissions required to perform their tasks.

---

## Objectives

- Create IAM Users
- Create IAM User Groups
- Attach AWS Managed Policies
- Create Custom IAM Policies
- Implement Least Privilege Access
- Validate Permission Restrictions
- Configure IAM Roles
- Understand Role-Based Access Control (RBAC)

# Lessons Learned

This project strengthened my understanding of AWS Identity and Access Management by demonstrating how permissions are assigned through users, groups, policies, and roles.

It also highlighted the importance of implementing the Principle of Least Privilege to improve cloud security while reducing unnecessary access to AWS resources.

---

# Key Takeaways

- IAM is the foundation of AWS security.
- Users should receive only the permissions required for their responsibilities.
- Groups simplify permission management.
- Managed Policies provide predefined access controls.
- Custom Policies allow fine-grained permission management.
- IAM Roles provide temporary credentials without exposing long-term access keys.
- Permission testing verifies that security configurations work as intended.
